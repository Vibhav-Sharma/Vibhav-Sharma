<details>
<summary>📖 &nbsp;<b>What are these papers about?</b></summary>
<br>

**DA-RAG** — Built a dynamic RAG system where queries are first clarified before retrieval. Uses SBERT + FAISS for retrieval and BART for generation. The clarification step reduces hallucination by making sure we fetch the *right* context, not just semantically close context.

**KV Cache Management** — LLMs waste GPU memory storing all token embeddings forever. This work proposes evicting tokens based on *semantic topic clustering*, so high-value tokens survive longer than random ones. Outperforms FIFO and LRU baselines.

</details>

<details>
<summary>⚙️ &nbsp;<b>One-time snake setup (2 min)</b></summary>
<br>

Create `.github/workflows/snake.yml` in your **`Vibhav-Sharma`** profile repo:

```yaml
name: Snake Animation

on:
  schedule:
    - cron: "0 0 * * *"
  workflow_dispatch:
  push:
    branches: [main]

jobs:
  generate:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v3

      - name: Generate snake SVGs
        uses: Platane/snk/svg-only@v3
        with:
          github_user_name: ${{ github.repository_owner }}
          outputs: |
            dist/github-snake.svg
            dist/github-snake-dark.svg?palette=github-dark

      - name: Deploy to output branch
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
          publish_branch: output
          commit_message: "Update snake animation [skip ci]"
```

Then: **Actions tab → Run workflow manually** → done ✅

> **Why it was broken before:** The old workflow used `crazy-max/ghaction-github-pages` which is deprecated and fails silently. `peaceiris/actions-gh-pages@v3` is the correct one.

</details>
